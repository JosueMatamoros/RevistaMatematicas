\section{Introducción}  
Un documento \tt{tex} se puede compilar de varias maneras. En nuestro caso queremos algo bimodal: Queremos como salida un  documento \tt{pdf} y un documento \tt{html} (para agregar contenido interactivo). \\

Para obtener un documento \tt{pdf}, compilamos con PDFLaTeX y para traducir el documento y obtener un sitio web, con su archivo de estilo \tt{css},  compilamos con Make4ht. El archivo \tt{css} se modifica y/o se amplia con un archivo personalizado  \wtt{config.cfg}. En la Figura \ref{fig:fig0} se muestra el proceso de manera simplificada.\\

\bigskip
Este documento está implementado con código que ejecutan tanto PdfLaTeX como Make4ht. Pero hay partes en que solo nos interesa una salida \tt{html} y otras en las que solo nos interesa la salida \tt{pdf}, para esto usamos comandos que PDFLaTeX ignora y que Make4ht sí compila y viceversa.\\


Make4ht se enfoca en la estructura del contenido y, si no hay errores de compilación fatales,  entrega un archivo \tt{html} correcto pero con un dise\~no simple. Este artículo es un ejemplo (y al mismo tiempo es una plantilla) de cómo agregar dise\~no web moderno y \tt{responsivo}, de manera manual, a través de un archivo de configuración y también a través de un framework como Bootstrap, \cite{bootstrap}. \\

Este documento considera las {\it tareas comunes} en la implementación de material didáctico en matemáticas y agrega un archivo de configuración para la salida moderna y correcta en Html. La manera de configurar Make4ht para lograr este objetivo se describe en la sección \ref{sec:configurarmake4ht}.\\

\fvnp{?`Qué nivel necesitamos?} Para usar la plantilla,  requiere ser un usuario normal de LaTeX y  posiblemente conocer cosas básicas de \tt{html} y \tt{css} (podría empezar por \cite{w3html} y \cite{w3css}). La parte interactiva requiere conocer JavaScript, pero pretendemos que el código lo genere la IA.\\

 Si quiere conocer cómo se configuró la plantilla y eventualmente modificarla, entonces se debe leer la sección \ref{sec:configurarmake4ht} y conviene leer \cite{bmake4ht}. \\









\fvnp{¿Por qué traducir un documento \tt{pdf}  a \tt{html}?} El formato \tt{html} ofrece accesibilidad, interactividad y un mayor alcance en el entorno digital actual. Generar el \tt{html} desde un archivo \tt{latex} permite aplicar revisiones al \tt{pdf} sin necesidad de tener que volver a editar manualmente el \tt{html}.\\

Mientras el \tt{pdf} es ideal para impresión y citación formal, el \tt{html} se adapta a cualquier dispositivo y permite integrar elementos dinámicos como visualizaciones, actividades interactivas y contenido multimedia. Este enfoque promueve la ciencia abierta y la educación digital al hacer el contenido más flexible y accesible.


\section{Make4ht para impacientes} Bien, tenemos una plantilla \wtt{doc.zip} con figuras y varios archivos \tt{tex} y un archivo de configración \wtt{config.cdf} (entre otras cosas) que  Make4ht traduce bien a \tt{html}.\\

 Podemos agregar paquetes y comandos personales, en el archivo \tt{ComandosdelUsuario.tex} que está en la carpeta "Paquetes". Pero hay que observar que {\it hay paquetes incompatibles} con Make4ht y posiblemente también comandos. Siempre hay que estar compilando para ir probando como funcionann las cosas.\\
 
 
 \tt{make4ht} es altamente configurable, por lo que  el diseño del sitio web  final lo podemos automatizar a través de un archivo de configuración que contiene mayormente código \tt{css}, para aplicar ajustes específicos para el formato de salida \tt{html}.\\
 
\fvnp{Sentido común.} Hay que pensar en "modo pdf" si visualizamos el \tt{pdf} de salida, y en "modo html" si visualizamos la página web de salida. Son mundos parecidos, pero {\it hay cosas en pdf que no tiene equivalente en html} y viceversa. Así que hay que estar valorando qué cosas tienen sentido en uno y en el otro ambiente. La buena noticia es que  tenemos dos comandos para saltar de uno a otro modo en este documento \tt{tex}: Las cosas sofisticadas en \tt{latex},  buenas para \tt{pdf} pero que podrían  no compilar bien con Make4ht, las podemos simplicar  y hacer una versión \tt{latex} más sencilla (pero equivalente) o inyectar código \tt{html} de manera directa.\\

Cosas demasiado complicadas se puede recortar e incluir como una imagen \tt{svg}. La manera automática de hacer esto lo vemos  en la sección \ref{tosvg} Prácticanmente todo el texto en una página web, está en formato vectorial \tt{svg}, porque este formato permite {\it renderizar} el texto en alta  calidad y permite escalar sin pérdidas.\\

 \subsection{Figuras}\label{figuras}
En \tt{html}, los formatos  mejor soportados son \tt{.png}, \tt{.jpg} y \tt{.svg}.


\be
\item \fvnp{png}. Incluir una figura \tt{png} no requiere hacer cambios, compila igual con PDFLaTeX y Make4ht.\\

\lstinline+\includegraphics[width=0.7\textwidth]{images/fig2.png}+\\


Para {\textcolor{wttD2}{escalar}} una figura \tt{png} (o \tt{jpg}) se puede usar la opción, \lstinline+[width=0.x\textwidth]+.  Esta opción, en términos de \lstinline+\textwidth+, es adecuada para \tt{pdf} y \tt{html}.\\

Estas figuras se inyectan en el  \tt{html} con algo como\\

\lstinline+<img alt='PIC' class='includegraphics' src='images/fig2.png' width='310' />+.\\

Es decir, también podemos insertar figras \tt{png} desde \tt{latex }. Por ejemplo,\\

\begin{lstlisting}
\insertarhtml{% Make4ht
   <img alt='PIC' class='includegraphics' src='images/fig3.png'
         width='150' />}
\end{lstlisting}


\item \fvnp{svg}. El formato \tt{svg} (Scalable Vector Graphics) se usa para mostrar gráficos vectoriales escalables y nítidos en la web.\\

Se puede incluir incluir una figura \tt{svg} con  Make4ht, usando el código\\

\begin{lstlisting}
\pdfomk{% PdfLaTex no soprta este formato
}{% Make4ht soporta:
\includegraphics{images/fig1.svg}
}
\end{lstlisting}

\bigskip
PDFLatex puede incluir figuras \tt{svg} con el paquete svg, pero solo funciona para PDFLaTeX.

\bigskip
Make4ht inyecta la figura \tt{.svg}  en el  \tt{html} con algo como\\

\lstinline|<object class='graphics' data='images/fig1.svg' type='image/svg+xml'></object>|


\begin{itemize}
\item \lstinline+<object>...,/object>+ es un elemento \tt{html} que incrusta contenido externo (\tt{svg, pdf, html}, etc.) como un "documento embebido".\\

 \item Estas figuras \tt{svg} {\it no se pueden escalar} con la opción usual \lstinline+[width=0.x\textwidth]+.\\
 El navegador web las inserta con un tama\~no absoluto. Esto se debe a que una figura \tt{svg} se puede escalar si accedemos a su atributo \wtt{viewBox} (sistema de coordenadas), pero Make4ht no hace eso, solo inserta la figura.\\

 \item Para escalar estas figuras se necesitan programas externos.\\

 \begin{itemize}
 \item Podemos escalar las figuras \tt{svg}  visualmente (o con comandos en terminal) por ejemplo con con \wtt{Inkscape}. \\

 \item En \wtt{Linux} se puede hacer en la terminal con \wtt{rsvg-convert}.\\

 Por ejemplo si queremos escalar \wtt{fig1.svg} a un ancho \tt{150pt}, ejecutamos la instrucción\\


 \begin{lstlisting}
  % sudo apt install librsvg2-bin
  cd images
  rsvg-convert -f svg -w 150 fig1.svg -o sfig1.svg
  % Windows
  % Usar Inkscape o un programa similar
 \end{lstlisting}

 \bigskip
 \wtt{sfig1.svg} es la figura escalada. Como el \tt{html} toma las figuras de la carpeta \tt{images}, podemos seguir ajustando al figura y recargando el \tt{html}.\\

 Hay comandos equivalentes para Windows y Mac.

 \pdfomk{}{
 \begin{tabular}{c|c}\toprule
 \fvn{Figura sin escalar} & \fvn{Figura escalada con \tt{rsvg-convert}} \\ \hline
  \includegraphics{images/fig2.svg} & \includegraphics{images/sfig2.svg}\\
  \bottomrule
  \end{tabular}
  \captionof{figure}{rsvg}
}

 \end{itemize}

\end{itemize}









\item \fvnp{pdf}. Los navegadores no {\it renderizan}  los figuras \tt{pdf} como imágenes (excepto con el elemento \wtt{<object>}). A veces  Make4ht puede convertir estas figuras automáticamente (en al compilación) a \tt{png} o \tt{svg}, mediante herramientas externas (como \tt{ghostscript} o \tt{pdf2svg}). Esto depende de si en el sistema están instaladas estas herramientas.\\
\ee

\section{Convertir  entornos "complicados" en imágenes {\color{tttxtD2}svg}}\label{tosvg}
Make4ht {\it no ofrece} soporte para muchos paquetes que se usan para implementar tablas o arreglos complejos, como las que se implementan con \tt{hhline}, \tt{nicematrix}, \tt{polynom} etc. Podemos  convertir estos entornos en imágnes \tt{svg} o también, si se presta, se podrían  convertir en \tt{html} interactivo (sección \ref{Webinteractiva}) \\

El paquete \tt{ltximg} \cite{ltximg}, se ejecuta con un comando {\it en la terminal}: Busca los entornos marcados con
\lstinline+%<*ltximg> %</ltximg>+ y los convierte en imágenes \tt{.svg}

\pdfomk{
\bc
\includegraphics[width=0.7\textwidth]{images/ltximg.pdf}
\captionof{figure}{\wtt{ltximg} convierte entornos a \tt{.svg}}
\ec}{
\bc
\includegraphics{images/ltximg.svg}
\captionof{figure}{\wtt{ltximg} convierte entornos a \tt{.svg}}
\ec
}


El entorno \lstinline+%<*ltximg> %</ltximg>+ es ingnorado por PDFLatex y por  Make4ht. . Pero \tt{ltximg} si lo interpreta como un {\it identificador}.\\

\begin{lstlisting}
%<*ltximg>
  Entorno...
%</ltximg>
\end{lstlisting}

\bigskip
Una vez que tenemos idenficados los entornos que vamos a convertir a imagen \tt{svg}, tenemos que {\it correr en terminal} el comando que recorta y hace las conversiones.\\


 \begin{lstlisting}
ltximg --skipenv 'tikzpicture' --runs 2 --subenv --svg --imgdir images -o doc-out doc.tex
 \end{lstlisting}

\bigskip
Funciona así: Este comando recorre el documento \tt{doc.tex} y localiza los entornos que están envueltos en el entorno  \lstinline+%<*ltximg> %</ltximg>+, luego genera y pega en la carpeta  \tt{images}, un archivo \tt{doc.tex} con solamente los entornos y el preámbulo del documento, luego genera la imagen  \tt{.pdf} de cada entorno  y las convierte a \tt{.svg}. Por supuesto: Como \wtt{ltximg} trabja en la carpeta images, debemos pegar temporalmente las carpetas donde tenemos las cosas del preámbulo.\\

Los nombres de las imágenes van numeradas por orden de aparición: \\

\wtt{conv-fig-1.svg}, \; \wtt{conv-fig-2.svg}, \;etc.\\

Para escalar las figuras \tt{svg} usamos \wtt{rsvg-convert} o \wtt{inkscape} o equivalentes, como se indicó en la sección \ref{figuras}.\\


\begin{nota}
También podemos hacer solo un \tt{.tex} aparte,  con solo  los entornos, los paquetes y los comandos necesarios y luego aplicar \wtt{ltximg} a este archivo.
\end{nota}

\section{Configurar Make4ht}\label{sec:configurarmake4ht}

%\tt{tex4ht} (implementado por Eitan Gurari (1947–2009)) es un conjunto de scripts que, después de la compilación con \tt{htlatex},  "traduce" el archivo
%\tt{dvi}. Este archivo \tt{dvi} contiene toda la información sobre la disposición del texto, gráficos y fórmulas matemáticas en un formato compacto. Esta información se usa para generar documentos \tt{pdf},  \tt{html}, etc.\\

%\tt{make4ht} es un sistema de compilación (implementado por Michal Hoftich) para \tt{tex4ht}.  \tt{make4ht} es una herramienta diseñada principalmente para convertir documentos \tt{LaTeX} en \tt{HTML}, enfocándose en la estructura del contenido (texto, tablas, imágenes, fórmulas matemáticas, etc.), pero {\it no se encarga del diseño web avanzado}.
